# 🤖 AI Coding Agents: คู่มือและบทบาทผู้ช่วยพัฒนาในโปรเจกต์

ยินดีต้อนรับสู่หน้าระบบการทำงานร่วมกันระหว่าง **คุณ (USER)** และ **Antigravity (AI Coding Assistant)** เอกสารฉบับนี้จัดทำขึ้นเพื่ออธิบายบทบาท หน้าที่ ตลอดจนแนวทางการทำงานร่วมกันระหว่าง AI Agents และการป้องกันโควตาทรัพยากรระบบเพื่อควบคุมค่าใช้จ่าย

---

## 👥 บทบาทของ AI Agents ในโปรเจกต์นี้

ในการพัฒนาและดูแลโปรเจกต์นี้ เรามีทีม AI Agents ที่มีคุณสมบัติเฉพาะตัวในการสนับสนุนงานดังต่อไปนี้:

| ชื่อเอเยนต์ (Agent Name) | บทบาทหน้าที่ (Role) | ความสามารถหลัก (Key Capabilities) | การเรียกใช้งาน (Usage Context) |
| :--- | :--- | :--- | :--- |
| **Antigravity (Master)** | ผู้ประสานงานและสถาปนิกหลัก | ควบคุมการแก้ไขโค้ด, พัฒนา UI, วางโครงสร้างสถาปัตยกรรม และจัดการ CI/CD | ดำเนินการตอบสนองคำสั่งหลักของ User โดยตรง |
| **Research Subagent** | เอเยนต์ฝ่ายวิจัยและค้นหา | ค้นหาโค้ดเชิงลึกในโปรเจกต์, สืบหาความรู้ทางเทคนิคจากภายนอก และสรุปผลวิจัย | ใช้เมื่อต้องการค้นหารูปแบบปัญหาที่ซับซ้อน หรือต้องการสรุปข้อมูลโดยไม่ต้องการรบกวน Context การทำงานหลัก |
| **Self Subagent** | เอเยนต์ทำงานคู่ขนาน (Clone) | รันคำสั่งย่อยในโครงสร้างแบบเดียวกับ Master เพื่อแบ่งเบาการทำงาน | ใช้ทำงานย่อยหลายอย่างพร้อมกัน เช่น การตรวจเช็คโครงสร้างไฟล์ควบคู่กับการเขียนเทส |

## 🛠️ คำสั่งทางลัดสำหรับการสื่อสาร (Slash Commands)

คุณสามารถสั่งการ AI ด้วยการใช้คำสั่ง **Slash Commands** เหล่านี้ในช่องแชทเพื่อเริ่มกระบวนการอัตโนมัติ:

*   **`/goal`**: ใช้เมื่อคุณต้องการสั่งงานระยะยาวที่ค่อนข้างซับซ้อน (เช่น การทำความสะอาดโค้ดทั้งระบบ) และต้องการให้ AI ดำเนินงานอย่างละเอียดแบบ Non-stop จนกว่าจะบรรลุเป้าหมาย
*   **`/schedule`**: ใช้เมื่อต้องการตั้งเวลาทำงานล่วงหน้าหรือตั้งเวลาทำงานซ้ำๆ (เช่น การสั่งให้ AI ตรวจสอบสถานะการ Deploy หรือตรวจสอบ Log การทำงานทุกๆ 1 ชั่วโมง)
*   **`/grill-me`**: ใช้เมื่อคุณและ AI ต้องการสัมภาษณ์แลกเปลี่ยนความเห็นเกี่ยวกับการออกแบบ (Design Decision) หรือเมื่อคุณต้องการวิเคราะห์ความเสี่ยงและทางเลือกต่างๆ ร่วมกันก่อนลงมือจริง
*   **`/teamwork-preview`**: ใช้เพื่อดูพรีวิวและจำลองการกระจายงานให้เอเยนต์หลายๆ ตัวเข้ามาทำงานร่วมกันแบบคู่ขนานสำหรับโปรเจกต์ขนาดใหญ่

## 🔒 แนวปฏิบัติสำหรับ AI Agents เพื่อรักษาโควตา (Free Tier Constraints)

เพื่อป้องกันไม่ให้การใช้งานทรัพยากรบน **AWS** และ **GitHub** เกินวงเงิน **Free Tier** ทีม AI Agents จะยึดถือแนวทางปฏิบัติอย่างเคร่งครัดดังนี้:

### 1. การประมวลผลบน AWS Lambda & API Gateway
*   **งดใช้ Scheduled Event ที่ถี่เกินไป**: ห้ามเปิดใช้งาน Cron job ของ TradeBot บน Lambda เว้นแต่จะระบุความถี่ที่ต่ำมาก เช่น ทุก 1 ชั่วโมง หรือปิดการทำงานถาวรเมื่อไม่ใช้งาน
*   **การประยุกต์ใช้ Rate Limiting**: นำ Middleware `throttle` เข้ามาครอบเส้นทาง (Routes) ทุกเส้นทาง เช่น หน้าหลักจำกัดที่ 30 requests/นาที และฟอร์มติดต่อจำกัดที่ 3 requests/นาที เพื่อป้องกันการโดนบอตยิงถล่ม (DDoS/Spam) ซึ่งจะส่งผลให้ค่า Lambda Invocations เกิน 1,000,000 ครั้ง/เดือน
*   **เลี่ยงการเขียนข้อมูลลง Disk/Database บ่อยครั้ง**: ใช้ระบบ Log หรือส่ง Event ไปที่ External Slack/Telegram/SQS แทนการเขียนลงไฟล์ SQLite บน Lambda (เนื่องจากระบบไฟล์เป็นแบบ Read-only และการสร้างไฟล์ใหม่บน `/tmp` จะส่งผลให้เนื้อที่เต็มหรือสูญหายเมื่อระบบ Cold Start)

### 2. การควบคุมโควตา GitHub Actions
*   **จำกัดทริกเกอร์ที่ไม่จำเป็น**: ตั้งค่า `paths-ignore` ในไฟล์ Workflow เสมอ เพื่อข้ามการรันกระบวนการ CI/CD เมื่อแก้ไขเพียงไฟล์เอกสาร เช่น `.md`, `.cursorrules` หรือไฟล์ที่ไม่เกี่ยวข้องกับการใช้งานระบบจริง
*   **การใช้ Cache**: ใช้ `actions/cache` สำหรับ Composer vendor และ Node modules เพื่อประหยัดเวลาการ Build ในแต่ละรอบให้เหลือต่ำกว่า 3-4 นาที
